Docker

基于容器化敏捷开发实践

为什么需要容器

在使用容器技术之前，我们可能会遇到的问题

水平伸缩能力

VM 方式部署的服务，难以对服务进行水平伸缩。

单一运行环境

同一台服务器上运行多个服务，难以做到隔离。

服务生命周期

生命周期难以控制，难以实现灰度发布和滚动升级。

环境的一致性

开发人员难以通过代码的方式保证多环境的一致性。

提供的能力

容器化技术为我们带来了更多云原生能力

服务编排

通过服务编排能力，可以实现简单的 Infrastructure as code。

渐进式扩展

从单机到 Swarm 或 Kubernetes，可以根据业务规模逐步升级，并且支持服务的注册、发现和健康检查。

云原生支持

基于容器编排和服务治理实现动态伸缩和动态负载，提高效率的同时降低成本。

可观测性

提供多种可观测性技术生态支持，实现无侵入式接入观测能力。

开始部署


            # 根据自己实际环境，设置期望放置项目的目录，例如我这里将项目放在~/payssion
            export PROJECT_PATH=~/payssion && mkdir -p $PROJECT_PATH
            # 将 Docker 运行环境克隆到项目目录中
            git clone git@bitbucket.org:payssion/docker.git $PROJECT_PATH/docker
            # 将项目代码克隆到本地项目目录中
            git clone git@github.com:payssion/payssion-api.git $PROJECT_PATH/api
            git clone git@bitbucket.org:payssion/payssion-admin.git $PROJECT_PATH/admin
            git clone git@bitbucket.org:payssion/payssion-pubservice.git $PROJECT_PATH/pubservice
            git clone git@bitbucket.org:payssion/payssion-dashboard.git $PROJECT_PATH/dashboard-api
            git clone git@bitbucket.org:payssion/payssion-public-tools.git $PROJECT_PATH/public-tools
            # 定义 PHP 版本环境变量，并拷贝 PHP 相关配置文件
            cd $PROJECT_PATH/docker && export PHP_VERSION=8.1
            # 拷贝 PHP 相关配置文件
            cp -R templates/php/$PHP_VERSION/etc/* services/fpm/etc/
            # 复制开发环境的 PHP 配置文件
            mv services/fpm/etc/php/php.ini-development services/fpm/etc/php/php.ini

服务编排


              # 复制对应环境的 Docker compose 编排配置文件
              cp docker-compose-local.yaml docker-compose.yaml

              # docker-compose.yaml
              include:
                - path: compose/volumes.yaml
                - path: compose/networks.yaml

                services:
                  fpm:
                    labels:
                    - dev.orbstack.domains=fpm.${TLD}
                    extends:
                    file: compose/fpm.yaml
                    service: fpm
                  nginx:
                    labels:
                    - dev.orbstack.domains=*.${TLD}
                    extends:
                    file: compose/nginx.yaml
                    service: nginx
                  redis:
                    labels:
                    - dev.orbstack.domains=redis.${TLD}
                    extends:
                    file: compose/redis.yaml
                    service: redis

              # 启动容器
              docker compose up -d

对于需要自定义服务内容的，可以使用 extends 方式引入，而无需自定义的服务，则直接在顶部 include 即可。

mDNS

OrbStack 使用 mDNS 为容器设置域名解析，仅支持 .local TLD


            # 查看域名解析
            dns-sd -Q redis.payssion.local

HTTPS

OrbStack 自动为容器开启 HTTPS（但目前尚不支持容器间 TLS 通信）


            # 安装 mkcert
            brew install mkcert
            # 生成所需的证书
            cd $PROJECT_PATH/docker
            mkcert \
              -cert-file services/nginx/certs/fullchain.cer \
              -key-file services/nginx/certs/fullchain.key \
              payssion.local "*.payssion.local"

作者在后续版本中会考虑支持容器间的 HTTPS 通信
